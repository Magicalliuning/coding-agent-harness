# V0.0.1 Runtime Baseline

Date: 2026-07-01

V0.0.1 closes the first vertical proof for the coding-agent harness runtime: a
CLI-started, PostgreSQL-backed, replayable self-recovery path that stops at
human commit approval.

## Scope

- Rust workspace and crate boundaries for core, events, database, policy, tools,
  models, context, runtime, and CLI.
- PostgreSQL migration baseline for runtime sessions and append-only EventLog.
- CLI session start, show, bounded context compilation, fake-model turn, small
  coding task, recovery fixture, and governed worker lane contract surfaces.
- Policy Gate and Tool Runtime boundary for safe verification command execution.
- Deterministic internal fake-model path that proposes a file patch without a
  real LLM dependency.
- Bounded Self-Recovery Loop for one fixture failure, with max rounds, repair
  budget, stop reasons, diff summary, token ledger, EventLog replay, and final
  `pending_commit_approval` state.
- Governed Codex CLI lane contract/schema/adapter boundary, without invoking a
  real external Codex worker in V0 acceptance.

## Acceptance Gate

V0 is accepted by the manual gate in
`docs/development/v0-acceptance.md`. The gate must be run against a copied
`fixtures/v0-acceptance` repository and a reachable PostgreSQL database.

The release verification set is:

```powershell
cargo fmt --all --check
cargo test --workspace
cargo build --workspace
cargo clippy --workspace --all-targets -- -D warnings
```

Manual acceptance command:

```powershell
cargo run -p harness-cli -- session recover-fixture $sessionId `
  --database-url $env:HARNESS_DATABASE_URL `
  --task "run the V0 acceptance self-recovery fixture" `
  --max-bytes 4096 `
  --focus agent `
  --max-recovery-rounds 2 `
  --max-repair-bytes 4096
```

Expected terminal evidence includes:

- `recovery_stop_reason=recovered`
- `verification_decision=allow`
- `verification_executed=true`
- `verification_exit_code=0`
- `diff_paths=.harness/fake-agent-turn.md`
- `event_replay_last=recovery.stopped`
- `final_state=pending_commit_approval`

## Source Of Truth

EventLog remains the runtime source of truth. Session state, recovery status,
approval state, tool observations, token ledger, and replay summaries are derived
from recorded events, not from a separate control panel or client-owned state.

## Known Limits

- No real LLM provider is wired into the V0 acceptance path.
- The Codex lane is a governed contract and adapter boundary, not a live cloud or
  external CLI execution lane yet.
- The harness does not automatically commit or push user code.
- PostgreSQL is required for the V0 runtime path.
- Web, iOS, IDE, MCP server mode, remote daemon control, and production k3s
  deployment remain outside V0.

## Next Tranche

After this baseline, new work should extend the runtime without changing the V0
source-of-truth boundary:

- real governed Codex CLI execution behind the existing lane contract;
- approval APIs and client surfaces over runtime projections;
- richer task queue and worker scheduling;
- runtime-backed Web, iOS, IDE, or MCP surfaces;
- hardened PostgreSQL migration and operational workflows;
- broader context compiler and eval coverage.
